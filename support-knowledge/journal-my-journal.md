---

## What is My Journal?

My Journal is a personal reflection space inside the Tree of Life Reader. It lets users write and save notes linked to their Tree Reader sphere readings, Transit prompts, and Oracle reflections.

The Journal does not cost Reader credits. Writing, saving, viewing, and exporting entries are all free.

---

## Where to find My Journal

- Open the Reader burger menu (top-right) and tap **My Journal**
- Tap **Open Journal** beneath any Tree Reader reading prompt
- Tap **Open Journal** beneath the Transit prompt
- Tap **Open Journal** at the bottom of the Oracle screen

---

## How entries work

- Each entry is linked to the reading or prompt where it was opened (e.g. Tree Reader: Malkuth, Transits, Oracle)
- Entries appear newest first
- After saving, an entry collapses to a marker line such as: `TREE READER: MALKUTH · 07/06/26`
- Tapping a saved marker reopens that entry for reading

---

## Journal Key

When a user opens My Journal for the first time, a Journal Key is generated. They should copy it and keep it safe.

**Safe wording for users:**
"Your Journal Key is created when you first open the Journal. Copy it and keep it safe. Recovery features are planned for a later encrypted backup phase."

Do not say the key is emailed. Do not say cloud recovery already exists.

---

## Where Journal entries are stored

Phase 1 Journal entries are stored locally in the user's browser on the device where they were written. They are not yet synced to the cloud or to the user's Stoic Qabalah account. They are not automatically available on other devices or browsers.

Entries can be lost if the user:
- Clears browser data or cache
- Uses a private or incognito browsing window
- Switches to a different browser
- Switches to a different device

**Recommend Export backup** to all users asking about Journal storage.

---

## Export backup

Export backup downloads a copy of the Journal as a file. The user should keep this file safe. This is the recommended way to protect Journal entries during the current local Journal phase.

If a user asks "What is Export backup?":
"Export backup downloads a copy of your Journal so you can keep it safe outside the browser. This is recommended during the local Journal phase."

---

## Does the Journal use credits?

No. Journal writing, saving, viewing, and export do not use Reader credits.

## Common support questions

**"Where is my Journal?"**
"You can open My Journal from the Reader menu, or use the Open Journal button beneath Tree, Transit, and Oracle prompts. Your entries are saved on this device and browser during the current local Journal phase."

**"Why can't I see my Journal on another device?"**
"The current Journal is saved locally on the browser and device where it was written. It is not yet synced across devices. To keep a copy, use Export backup from the Journal menu."

**"Where did my Journal go?"**
Ask the user:
1. Are you using the same browser and device where you wrote your entries?
2. Have you cleared browser data or cache recently?
3. Were you using a private or incognito window?
4. Did you export a backup?

If browser data was cleared and no backup exists, Star Support cannot recover local-only entries.

**"Can Star Support recover my Journal?"**
"Journal recovery is not yet cloud-enabled. If you exported a backup, keep that file safe. The current Journal is saved locally on the device and browser where it was written. Encrypted cloud backup and recovery is planned for a later phase."

**"Can I recover my Journal?"**
Same answer as above. Do not promise recovery unless the user has a backup file or cloud recovery has been implemented.

**"Why did I receive a new Journal Key?"**
Possible causes: different browser or device, browser storage cleared, private or incognito window, or the app was opened in a fresh browser context. Advise the user to use the same browser and device consistently, and to export a backup regularly.

**"Does my Journal sync between devices?"**
"Not yet. The current Journal is saved locally on the browser and device where you write it. To keep a separate copy, use Export backup."

**"Does writing in the Journal use credits?"**
"No. Journal writing, saving, viewing, and export do not use Reader credits."

---

## Boundaries

- Star Support cannot directly read, view, or recover a user's local Journal entries.
- Do not promise cloud recovery — it is not active in Phase 1.
- Do not expose internal localStorage key names or implementation details.
- If a backup file exists, advise the user to keep it safe. Restoration from backup is done through the Recover my Journal option in the Reader menu.
